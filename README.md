Description
  A Book Borrowing System is a web-based application designed to digitally manage the borrowing and returning of books in a library or similar institution. It replaces manual record-keeping with an organized, efficient, and accessible platform for both administrators and members.

**📍System Flow (Overview)**
1. Admin visits and logs into the system
System displays the login page prompting for username and password
Credentials are authenticated against the users table
If invalid, an error message "Invalid Credentials" is shown and the admin is returned to the login page
On success, a session is started and the admin is redirected to the dashboard
2. System displays the Admin Dashboard
Displays an overview of the book borrowing system
Admin selects an action to perform from the dashboard
3. Admin manages books
Add new books with details such as title, author, category, and quantity
Edit or update existing book information
Delete books from the catalog
System prevents deletion if the book has an active borrowing record
All changes are saved to the database
4. Admin manages members
Add new member accounts with the required information
Edit or update existing member details
Delete member accounts from the system
System prevents deletion if the member has an active borrow record
All changes are saved to the database
5. Admin views and manages borrow records
View all active and historical borrowing transactions
Monitor due dates and identify overdue books
Update or correct borrow records as needed
All changes are saved to the database
6. Admin logs out of the system
Session is destroyed
System redirects back to the login page
